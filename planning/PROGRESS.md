## Recent Changes

- **Phase C/D Editor: Dialog Editor**: Visual node-graph dialog editor integrated into the dungeon editor as a separate mode. **New files**: `DialogEditorState.ts` (state model — tree CRUD, node rename with ref updating, validation, undo/redo snapshots, dirty tracking), `DialogGraphCanvas.ts` (canvas node graph — pan/zoom, drag nodes, bezier connections with arrowheads, color-coded start/selected/broken nodes, double-click to add, right-click context menu), `DialogInspector.ts` (right-panel node editor — ID/speaker/text fields, choices vs linear mode toggle, collapsible choices with preview headers, condition sub-editor per type, effect sub-editor per type, NPC sprite at top), `DialogNodeLayout.ts` (BFS auto-layout when no sidecar exists), `dialogIO.ts` (API client for dialog + layout files). **Modified**: `vite.config.ts` (3 dialog API routes — list/load/save with layout.json support), `editor.html` (dialog canvas + inspector containers, `.tool-btn:disabled` style), `EditorApp.ts` (`editorMode` + `dialogNpcId` fields), `Inspector.ts` ("Edit Dialog" button on NPC entities), `main.ts` (mode switching — enter/exit dialog mode, toolbar button hide/show with state preservation, "Add Node" + "Save Dialog" buttons in toolbar, dialog undo/redo/save keyboard shortcuts, context menu wiring, `updateDialogStatus` light refresh). **Features**: expandable sections preserve state across refreshes (conditions/effects don't collapse on edit), `+ new node` option in next dropdowns, condition/effect type switch sets default field values, collapsible choice entries with numbered preview text, validation errors in banner (broken refs, unreachable nodes, empty text, missing startNode), sidecar `.layout.json` files for node positions. 725 tests.
- **Dialog Overlay Keyboard Navigation**: Arrow Up/Down to navigate choices with visual highlight, Enter to confirm highlighted choice, Escape always ends conversation. Hint text updated. `onDismiss` callback for forced close.
- **Phase C: Quest System**: Data-driven quest system with JSON quest definitions, runtime state machine, rewards, quest log overlay, and save/load persistence. `questManager.ts`: QuestManager class with `loadQuest()` (fetch + cache from `/data/quests/`), `startQuest()`/`advanceQuest()` state machine (undiscovered→active→complete), reward application (xp, gold, items, flags), serialization for save/load, `installConditionEvaluator()` replaces `questStage` stub in dialogManager. `questLogOverlay.ts`: DOM overlay (J key toggle, Escape to close), dark dungeon panel (#2a1a0a, gold border), active quests with stage description + dimmed quest description, completed quests with checkmark, rebuilt on each show(). `dialogManager.ts`: `setConditionEvaluator()` export with separate `customEvaluators` map (custom evaluators shadow defaults). `saveSystem.ts`: optional `quests` field in SaveData + BuildSaveDataParams/ApplySaveDataResult threading (backward-compatible). `main.ts`: quest loading at init, condition evaluator wired, dialog hooks replaced with real quest calls (startQuest shows quest name, advanceQuest shows complete/updated), J key opens quest log, overlay guards + anyOverlayOpen, save/load quest state. 3 quest data files (fetch_amulet, kill_spider_queen, collect_lore) — single-stage design matching dialog-driven advancement (one advanceQuest call per quest). `amulet_gregor` item added to items.json for fetch_amulet quest testing. 725 tests (30 new questManager tests).
- **Phase A/B Editor: NPC Editor Support**: Full editor integration for the NPC entity type. Toolbar: teal circle with "NPC" label in entity palette. GridCanvas: teal circle icon on grid, sprite preview from npcDatabase when Item Preview enabled. EditorApp: `npc` entity defaults (`npcId: ''`). Inspector: npcId dropdown with sprite swatches (mirrors enemy dropdown pattern), readonly details panel (name, dialog file, merchant stock), sprite preview. editor/main.ts: `npcDatabase.load()` at init. Red hover highlight on invalid entity placement cells (all entity types, not just NPC — checks `canPlaceEntityType` and shows red for walls/invalid cells). NPC facing field removed from editor, game code, level loader validation, and DUNGEON-DESIGNER.md (enemies don't have facing either; may revisit if front/back/side sprite system is added later). 695 tests.
- **Phase F: The Architect's Tomb**: 3-level dungeon (`public/levels/architects_tomb.json`) showcasing all M2 features. Level 1 "The Antechamber" (brick, tutorial — signs, tripwire→dart trap, trigger, locked door + key, lever + mechanical door, secret wall, rats + goblin). Default dungeon changed from `test_m2d.json` to `architects_tomb.json`. 695 tests.
- **Phase E: Save/Load System**: Full game persistence via localStorage. `saveSystem.ts`: `SaveData` model (version, player state, entity registry, level snapshots as Records, mutated grids), `serializeLevelSnapshot`/`deserializeLevelSnapshot` (Map↔Record, Set↔array), 5 manual save slots + 1 autosave, JSON export/import. `saveLoadOverlay.ts`: DOM overlay (dark dungeon theme, capture-phase keydown), slot rows with timestamp/name/level, Save/Load/Delete/Export/Import buttons, death mode variant with Restart fallback. `gameState.ts`: `getPlayerState()`/`restorePlayerState()`, `getPickedUpKeys()`/`restorePickedUpKeys()`. `main.ts`: `saveGame()`/`loadGame()` functions, Escape key opens save/load overlay, auto-save on stair transitions, death → load overlay if saves exist else restart, overlay pauses game loop.
